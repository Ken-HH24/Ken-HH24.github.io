组件实践 - DraggableTree可拖拽树形控件

Posted on 2021-12-13

preview

在上一次组件实践中模仿了 semi design 完成了 Tree树形控件 的基本功能。在这基础上可以添加许多额外功能，比如 select ，checkbox 的支持。但是这些都稍显简单了，基本思路就是在 TreeNode 里添加相应的事件监听，然后根据 F / A 架构在 TreeFoundation.js 里书写逻辑，添加对应的 selectedKeys 或者 checkedKeys 即可。

而拖拽功能在如今前端里应用得很广泛，semi design 在 Tree树形控件 里也集成了该功能。在我看来，实现的难点主要为：

  1. 在拖拽事件结束后，原节点顺序被打乱，如何快速得到新的节点顺序是关键
  2. 如何在拖拽的过程中提供较良好的用户体验，比如在用户拖动到文件夹上时，该结构会自动展开。又或者添加适当的提醒让用户知道现在的拖拽结果

而以上问题都在阅读 semi design 源码，并自己动手实践后得到解决。

draggable

首先要达到可拖拽的效果，就需要使用 draggable 这个 API ，首先需要显式地在标签里添加 draggable = ‘true’ 而不能只写 draggable 。

然后为每个 TreeNode 添加 onDragStart ， onDragEnter 等事件。其中在 onDragStart 事件中，需要特别地调用 e.dataTransfer.setData('text/plain', ''); ，否则在某些浏览器内不会生效。另外在拖拽 <li> 标签的过程中会发现鼠标状态一直是一个禁止的图标，非常影响用户体验，要解决这个问题则需要在其他 drag 事件中添加 e.dataTransfer.dropEffect = 'move'; ，改为移动图标。

F / A

首先依然是贯彻 semi design 的 F / A 架构，将整个事件的处理逻辑放在 TreeFoundation.js 中，然后将对应的事件处理函数放入 Context ， TreeNode 只负责调用即可。

onDragStart

在拖拽的最开始，需要做的事情主要有两步：

  1. 记录被拖拽的节点
  2. 更新 Tree 状态

主要涉及以下状态：

state function
dragging 整个 Tree 是否处于拖拽状态
dragNodeKeys 被拖拽所有节点的keys

onDragEnter

该事件的对象不再是被拖拽的节点，而是拖拽的目标位置，因此又有新的状态需要记录：

state function
dragOverNodeKey 当前拖拽到的节点 key
dropPosition 关于 dragOverNode 的位置，1表示上面，-1表示下面，0表示节点内部

而整个事件处理过程主要分为如下几步：

  1. 通过 dragNodeKeys 判断，如果当前的 dragOverNodeKey 为被拖拽目标，那么直接 return ，并清空上述的两个状态
  2. 使用 setTimeout 对上述两个状态进行更新，然后还需要根据 dragOverNode 对节点进行展开，该逻辑在上一次 onNodeExpand 函数里已经实现过一次了，所以进行复用即可

按照我的理解，在这里使用 setTimeout 是因为用户拖拽动作可能很快，频繁触发 onDragEnter 会导致 expand 工作立刻执行，造成页面闪动。把逻辑放入 宏任务 里，让取消之前的 expand 工作成为了可能。

calcDropRelativePosition 函数则是计算当前拖拽位置相对于 dragOverNode 在哪里的函数，按照 semi design 源码的逻辑，设置了阈值为 0.4 ：

  1. [0 - 0.4] * height ：处于节点上面
  2. [0.4 - 0.6] * height ：处于节点内部
  3. [0.6 - 1.0] * height ：处于节点下面

onDragEnd

该事件意味着拖拽结束了，对象是 dragNode 。函数会将包括 dragOverNodeKey 等状态清空。至于最后的 treeData 状态整理，会交由 onDrop 处理，所以此时大部分关于 drag 的状态已清空。

onDrop

该事件为某拖拽目标落到了某个元素上，事件的对象是该元素。此时需要做最后一步工作，就是根据拖拽结果整理新的 treeData 并重新渲染。而 semi design 内部没有这一部分逻辑，那是因为他将处理权交给了用户，在 storybook 里可以看见传入的 onDrop 函数，代表最一般的 treeData 整理逻辑。

在这里把逻辑写在了 updateTreeDataAfterDrop 函数里：

  1. 对 treeData 进行拷贝，找到 dragNode 的位置，在拷贝数组里删除，保存到另外一个变量 dragObj 里
  2. 判断 dragNode 是否落入 node 内部，如果是，那么直接在 node.children 末尾添加 dragObj 即可
  3. 然后判断是否落入 node 的底部，以及 node 是否展开，如果是，那么将 dragObj 放入 node.children 的第一位
  4. 最后根据 dragPosition 放入 node 的前一位或者后一位

cls

小结

此次实践在上一次的 Tree树形控件 上添加了拖拽功能，通过阅读 semi design 的源码又收获了不少：

  1. draggable api 和 onDragStart 等事件 应该如何应用
