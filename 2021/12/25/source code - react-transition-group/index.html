updateStatus

Transition 整个动画的过程就是由 updateStatus 驱动的，该函数主要做如下几件事：

  1. 取消储存的回调函数
  2. 根据 nextStatus 执行入场或退场动画的函数
  3. 检查 unmountOnExit 和当前状态是否为 EXITED ，如果是，更改状态为 UNMOUNTED

入场动画

performEnter 函数的职责就是要合理安排入场动画的阶段，首先需要从 props 中取出各个阶段的回调函数以及动画时间 timeout ，然后按顺序做如下几件事：

  1. 执行 onEnter 回调函数
  2. 设置状态为 ENTERING
  3. 执行 onEntering 回调函数
  4. 在 timeout 毫秒后，设置状态为 ENTERED
  5. 执行 onEntered 回调函数

CSSTransition

在平时的生产环境中很少会使用到 Transition ， 因为从前面的代码可以看出，其实现的只是状态的改变，并没有真正改变 DOM 的 className 。实现这一步骤的是 CSSTransition ，该组件建立在 Transition 的基础上。

classNames

CSSTransition 提供 classNames 这个 API ，然后用户书写不同的 CSS 来反映不同的动画阶段，阶段主要分为：

  1. classNames-enter , classNames-exit

SwitchTransition

SwitchTransition 负责维护两个组件切换之间的动画，提供两种模式， in-out ：组件二进场后组件一再退场； out-in ：组件一退场后组件二再进场。组件的状态分为 ENTERED ， ENTERING 和 EXITING

整体实现的逻辑为：

  1. 使用 current 缓存上一个 children 组件
  2. 在 getDerivedStateFromProps 生命周期内用 areChildrenDifferent 判断子组件是否变化，如果变化， status 更改为 EXITING
  3. 按照 mode 的不同策略渲染第一阶段动画，并将 status 更改为 ENTERING
  4. 按照 mode 的不同策略渲染第二阶段动画，并将 status 更改为 ENTERED

TransitionGroup

TransitionGroup 的职责是维护一组动画列表，在列表中某一项被添加或删除的时候，呈现其入场或离场动画。

那么思路也很简单，在组件的 state 内用 children 变量缓存上一次的动画列表，在每一次 render 之前，借助 getDerivedStateFromProps 生命周期对比两次的列表异同，然后为对应的变化子项添加动画

getInitialChildMapping 和 getNextChildMapping

这两个工具函数是获取 children 的映射关系，getInitialChildMapping 用于组件的初次渲染：

getNextChildMapping 函数的处理则复杂一些，也是功能实现的关键，主要分为如下几步：

  1. 获取最新的 children 映射
  2. 通过 mergeMappings 函数对比前后两次的 children 映射，获得合并的映射结果
  3. 遍历合并映射集的 key ，通过判断 key 是否存在于前后两个映射集，来为对应子项添加动画
  4. 如果是新子项，那么通过 in: true 即可实现入场动画
  5. 如果是旧子项，不仅需要设置 in: false ，还要在 onExited 钩子函数里把对应的子项从 TransitionGroup 里删除

FLIP

这个时候 TransitionGroup 已经完成了基本功能，但是会发现在子项动画退出后列表更新时会有断裂感，而如果之前用过 vue 里的 Transition 组件就知道，可以通过 v-move 完善列表的过度效果，官网里是这样写的：

这个看起来很神奇，内部的实现，Vue 使用了一个叫 FLIP 简单的动画队列，使用 transforms 将元素从之前的位置平滑过渡新的位置。

重点就在于 FLIP 这个概念，改词是 First ， Last ， Invert 和 Play 的简称，其叙说的其实是一种实现动画的思想。

在浏览器 DOM 发生改变，在 React 执行 render 的时候，里面元素的位置已经更新，可以通过一系列 API 获得元素的最新位置，但是此时浏览器还没有对元素进行渲染，那么在这个空隙里，就可以通过动画来达到平滑过渡的效果。

所以要实现 FLIP ，就需要对元素的各个位置进行缓存，也就需要通过 ref 来访问真正的 HTML 元素，这也是为什么 CSSTransition 需要用 React.forwardRef 来转发 ref ，接下来改写一下 utils 文件夹下的 getChildrenMapping 函数

在使用 React.Children.forEach 遍历的时候，在 callback 的基础上添加 ref 回调函数，直接调用 getBoundingClientRect() 将位置信息保存在 rectMap 中：

现在持有了节点的位置信息，那么就需要在 componentDidUpdate 这个生命周期函数内应用 FLIP 这个概念：

  1. F: 通过 lastRectMap 获取上一次节点的缓存位置
  2. L: 通过 this.state.rectMap 获取这一次节点的缓存位置
  3. 遍历 currentRectMap 的所有 ref
  4. I: 通过 getBoundingClientRect() 的 left 和 right 计算出偏移量
  5. 通过前面计算的 invert 构造 keyframes 和 options
  6. P: 直接调用 ref.animate() 启动动画

现在 TransitionGroup 就有了平滑的列表过渡效果，但是也存在一个问题，当上一个动画还没有完成的时候，如果此时列表又发生了变化，则会带上上一个动画，再次重复一遍，这一点感觉可以通过在 lastRectMap 标记正在进行动画的节点来解决，这里就不再实现了。

另外，在众多关于 FLIP 和 React 的文章中，其实都是建议使用 useLayoutEffect 来实现的，因为该 hook 明确了是在 DOM 更新后，浏览器渲染之前执行的。只不过这次使用的是 Class Component ，声明周期 componentDidupdate 同样发生在 render 函数之后，所以就以此来代替了。

小结

通过这次 react-transition-group 的实现，收获还是挺多的，主要是熟悉了动画过程的实现：

  1. 如何对动画划分阶段，通过 setState 和 setTimeout 里的回调函数进行过渡，然后在不同阶段增添 className 实现动画
  2. 在 SwitchTransition 和 TransitionGroup 里都明显使用了缓存变量，然后通过 getDerivedStateFromProps 来对状态进行划分，这样能保存组件的上一个状态，并以此采取不同的策略
  3. FLIP 动画思想，在 useLayoutEffect 或者 componentDidUpdate 里对前后 DOM 变化进行动画过渡。并且利用 animate 这一 API 实现
