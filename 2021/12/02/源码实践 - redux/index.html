源码实践 - redux

Posted on 2021-12-02 Edited on 2021-12-10

preview

redux 是著名的 JavaScript 状态管理库，翻看 github 的源码会发现，核心代码的实现简洁而又巧妙。通过自己敲代码实践可以加深对核心思想的理解。

createStore

dispatch

既然是负责状态管理，那么 redux 就应该有一个 store 来存储数据。store 里面的数据不可直接改变，而是委托一个 dispatch 函数来完成，通过派发 action ，交由 reducer 来产生新的状态 state 。

因此 createStore 函数的参数包括：

  1. reducer - 接收 action ，处理修改 state 的逻辑
  2. initialState - state 的初始值

subscribe

有的时候想监听 state 的变化，可以设计一个 subscribe 函数将回调函数保存，然后在执行 dispatch 方法的时候触发回调函数。

另外 subscribe 函数还应返回一个 unsubscribe 函数，方便解除监听。

combineReducers

至此基本的状态管理已经完成了，但是面对大型项目的时候，所有数据都会汇聚在一个 state 和 reducer 里面，造成代码臃肿。所以要使用键值对将不同部分的状态拆分开来。

设计一个 combineReducer 函数，该函数可以将多个 reducer 合并成一个，返回一个 finalReducer 。具体做法是：

  1. 接收一个 reducers 对象，该对象形式为 [key: string]: reducer
  2. 返回一个新的 reducer 函数，在接收 action 的时候，遍历所有 key ，根据 key ，找到对应的 oldStateForKey 和 reducer ，然后执行 reducer(oldStateForKey, action) ，得到 newStateForKey
  3. 在遍历过程中使用一个 hasChanged 变量判断状态是否有改变，如果没有改变，直接返回原来的 state

总结

通过阅读 redux 源码，对 redux 的工作原理有了一个更加清晰的认识，更重要的是有以下知识点的收获：

  1. middleware 的形态以及运行方式
  2. next 与 compose 函数的妙用，通过参数提升到外层函数解决运行时绑定问题
  3. subscribe 函数返回 unsubscribe 的写法，有利于程序健壮性
