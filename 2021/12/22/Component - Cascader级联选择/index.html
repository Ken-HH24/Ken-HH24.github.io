Component - Cascader级联选择

Posted on 2021-12-22 Edited on 2021-12-23

preview

偶然在公众号里看到一片文章，是关于 cascader 多选级联实现的，里面谈到 ant design 的级联选择没有提供多选的功能。本来想照着文章实现一下的，后来去看了一眼 semi design ，发现官方已经实现了 multiple 多选的 api 。因此去翻了对应的源码，并自己实现了一下。

  • https://github.com/Ken-HH24/Ken-HH24.github.io/tree/demo-typescript/src/components/Cascader

Select

翻开 semi design 的源码可以发现，在 render() 函数里主要有两部分组件被渲染了，一个是 Select ，另外一个是 Popover 。先从简单的 Select 讲起，也就是选项框。由于太简单，官方甚至没有将其封装成一个组件。

选项框的职责主要有两个：

  1. 用户点击负责改变 isOpen 状态，控制级联选项是否可见
  2. 用户确定选项之后，把最终内容反映在选项框内

实现第一个需求很简单，就是添加 onClick 事件通过 setOpen 来改变状态。而对于第二个需求，则是通过传入 content ，将 content 放入选项框内实现，以应对不同的结果渲染要求：

基本定义

与 Tree 树形控件的思路一致， Cascader 会接收 TreeData 作为参数，第一步就是先把其转换为 [key: string]: entity 的形式储存在 keyEntities 中，方便以后在操作中通过 key 找到对应的节点，而对于 TreeData 与 Entity 有以下定义：

转换的思路主要是通过 traverseDataNodes 函数对所有节点进行遍历，每一次访问到一个节点的时候，通过递归函数保留其父节点，然后根据父节点对应扩展该节点的信息，并转换为 BasicEntity 类型。

转换完成后，会执行 convertDataToEntities 函数传给他的 callback 函数，在回调函数内可以访问到刚刚转换出来的 BasicEntity ，然后将该值保存在 keyEntities 内：

单选

首先实现单选的需求， CascaderItem 是显式选项的组件，其接受 data 并会将其以列的形式进行分层次地渲染，那么第一个问题就是：在 Cascader 里将什么数据作为 data 传入？

如果直接将 treeData 传入，那么会损失掉之前使用 convertDataToEntities 函数获得的信息，而如果直接将 keyEntities 传入，则会失去原有的树形结构。 semi design 的做法是通过判断 Entity 的 parentKey 是否存在，只将第一层的数据传入：

然后就有第二个问题：只传入了第一层数据，在用户点击后，怎么展示对应的第二，第三层数据？

semi design 会维护一个 activeKeys ，在用户点击某个选项后，会将该选项的 key 保存在 activeKeys 里，然后在 CascaderItem 里渲染时，判断节点是否为 active 状态，如果是，可以通过 BasicEntity 里的 children 属性直接获得下一层渲染节点

另外，除了 activeKeys ，当然还要保存现在被选择的节点，所以需要使用 selectedKeys 来实现，在点击事件处理函数中，需要多添加一段判断，如果被点击的节点已经处于 selected 状态，那么需要将其 key 移出 selectedKeys ，达到取消选择的效果。

可以控制选项的显示进度了，那么接下来要实现的就是最后的结果呈现了。单选模式下，当用户点击到叶子节点时，选项框应该自动收起，并在 select 里显示用户的选择结果。

这一部分逻辑依托于 selectKeys 。在前面的点击事件处理函数里有一个缓存变量 lastSelectedKey 。该变量在此时就发挥作用了，判断是否为叶子结点，如果是，则关闭选项框，并且设置选项框里的内容：

多选

有了前面的单选基础，实现多选起来就轻松许多，依然是相同的思路，使用 checkedKeys 维护。不过需要注意两点：在事件处理函数里，如果点击的节点拥有子节点，那么需要将其所有子节点都选上或取消。而如果同级所有节点都被选上或有其中一个不被选中，其父节点也要同时更新状态：

当 checkedKeys 改变后， select 里的 content 也需要改变，只不过与单选不同，多选显示的是一个个标签，标签有点击取消选中的功能。所以需要派生出另外一个组件 SelectItem ：

另外注意， semi design 里实现了自动合并的功能，也就是当一个节点的所有子节点都被选中时， select 中只会显示该节点，而不会显示其子节点，要做到这一点也很简单，在得到 content 的时候通过 Entity 的 parentKey 属性判断就好：

小结

要完成基本的 Cascader 需求并不难，关键是如何设计组件的结构，主要的收获在于：

  1. 组件结构， semi design 是借助 Popover 来实现弹框的
  2. 回顾了一遍 keyEntities 的转换过程，并根据组件需求来设计数据结构
  3. 整个组件的效果依然是由 activeKeys ， selectedKeys 等几个 Set 驱动的
