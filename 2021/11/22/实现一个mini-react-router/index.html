Posted on 2021-11-22 Edited on 2021-11-23

BrowserRouter

history & location

react-router 库中的 BrowserRouter 是利用 history 这个 api ，通过监听 popstate 动作实现的。

history 对象可以利用 createBrowserHistory() 这个方法获取，然后用 useMemo 缓存起来。

location 则使用 useState 保存，方便进行监听变化后重新触发渲染。

RouterContext

需要创建一个 context ，方便子组件访问到上述属性

BrowserRouter.js

Switch

Switch 组件的任务是获取当前的 location，然后匹配目标路由组件，进行选择性的渲染。

通过 React.Children.forEach 这个方法可以遍历所有子组件，获取子组件的 path 属性，与 location.pathname 进行比较，找到匹配的组件，并且添加 computedMatch 这个属性

Route

Route 组件代表匹配到对应 path 后，渲染目的组件。而组件的传入方式有多种，可以通过 children ，component 或者 render 函数等传入

newRouterContext

首先要根据组件的 path 属性和 location.pathname 比较，创建一个新的路由上下文环境

当父组件有 Switch 包裹的时候，如果是匹配的组件，可以从 props 中获取 computedMatch 属性。

如果没有，则可以通过 matchPath 这个 api 来对 location.pathname 和 props.path 进行比较，然后形成新的 路由上下文

hooks

useHistory & useLocation

当不想使用 props 或者 context 等方式访问 history 和 location 这两个变量时，react-router 还提供了钩子函数。

因为这两个变量存储在 context 中，所以直接使用 useContext 访问即可

useListen

有的组件可能存在于 context 包裹之外，但是依然想监听 history 的变化，可以设计一个钩子函数做到这一点。

在前面使用 createBrowserHistory() 创建 history 对象的时候用了一个 rootHistory 变量保存下来，并 export 出去。所以在该钩子函数中，就可以通过这个变量来注册监听事件，并执行对应的回调函数。

HOC

withRouter

在 react-router 中还用 withRouter 这个函数对组件进行处理，目的同样是将 history 等属性注入到组件当中。

基本思路就是通过 useContext 获取 routerContext，然后返回一个新的 Component，将 routerContext 注入，同时记得要转发 ref。

Link

Link 本质其实就是一个 <a> 标签，只不过他的跳转能相应地触发对应 history 的 popstate 事件。

利用前面实现的 useHistory 获取对应 history 对象，然后执行 push 动作即可
