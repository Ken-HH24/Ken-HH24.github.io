fre源码阅读 - reconcile篇

Posted on 2021-11-27 Edited on 2021-11-28

preview

上篇的 schedule 模块为 fre 构建了完整的任务调度系统。而当要真正执行更新的时候，就得从 reconcile 函数看起，因此写一个简单的应用，并触发 setState ，看一下 reconcile 模块是如何工作的。

reconcile

reconcile 函数接收一个 Fiber ，然后会不断执行 capture 函数进行深度优先搜索更新。
函数终止的情况有以下两种：

  1. 更新完毕，WIP 变量为 null ，开始进入下一个 commit 阶段
  2. shouldYield() 函数返回 true，调度结束，利用 bind 函数保存当前的更新进度。

capture

capture 函数会先判断当前的 Fiber 是否为 Functional Component，然后保存在 Fiber.isComp 中。根据判断结果分别执行 updateHook 或者 updateHost
因为是深度优先搜索，所以会先查看是否有 child，如果有，直接返回到上一层的 reconcile 函数，继续更新。
否则先查看是否有同层兄弟节点 sibling，并视情况返回。如果都没有，则重新指向 parent。

updateHook

如果当前 Fiber 是 Functional Component，那么会执行 updateHook。
该函数会执行以下任务：

  1. 重置 cursor 指针，该指针在生成 Fiber.hooks 中有作用
  2. 重新执行 Fiber 的函数，即 Fiber.type(Fiber.props)
  3. 进行 diff 算法

diffKids

diff 算法的流程主要如下

  1. 从尾指针开始进行 diff 操作

  2. 从头指针开始进行 diff 操作

  3. 如果旧节点已经遍历完，那么剩余的新节点都作 insert 操作

  4. 如果新节点已经遍历完，那么剩余的旧节点都作 remove 操作

  5. 在剩余节点之中，按照顺序找到最多的可复用节点，然后 clone 到新节点里。下面详细说一下实现过程。

algorithm

首先会得出如下变量：

  1. I对象 维护 [ newNode.key : newNode.index ] 的对应关系
  2. P数组 存储 P[newNode.index] = oldNode.index 的关系

随后调用 findLis() 函数，传入 P数组 ，查找其最长上升子序列。

LIS 的求解算法是经典的二分查找，思路如下：

  1. 顺序遍历，当前的数为 x
  2. 通过二分查找找到 seq 数组里比 x 小，但最大的数，记录其位置 pos （seq数组升序）
  3. 如果 pos+1 >= seq.length 或者 seq[pos+1] > x ，则 seq[pos+1] = x
  4. 否则继续下一次循环

linke

在前面的源码中多次看到了 linke 这个函数，该函数的作用是在重新构建 Fiber 树的过程中，维护 Fiber 的 sibling 和 prev 变量。
这里的 Fiber 树有一些不一样，头结点是 kids[kids.length -1] ，然后通过 sibling 指针一直指向尾节点 _prev = kids[0] 。

总结

reconcile 模块主要负责 Fiber 树重建工作，整体工作流程如下：

  1. reconcile 函数不断调用 capture 函数，如果调度结束，通过 bind 函数保存当前工作进度。
  2. capture 函数进行深度优先搜索，按照 Fiber 的类型执行更新。如果是 Funtional Component 会重新执行一次函数。
  3. capture 搜索过程中还会将 Fiber.hooks.effect 放入调度队列中。
  4. 每次 Fiber 更新执行 diffKids 函数，利用 diff 算法找到可复用的节点，其中用到了 LIS 和 二分查找 等常用算法。
  5. 搜索完毕使用 finish 保存最终 Fiber，进入 commit 流程。
