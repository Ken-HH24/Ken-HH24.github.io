实现一个Modal弹窗组件

Posted on 2021-11-26 Edited on 2021-11-27

弹窗组件的实现看起来并不困难，只需在组件内添加一个容器，一个背景蒙版，然后通过 visible 变量来控制显示即可。

但是这次实现的弹窗组件不局限于简单的功能需求，还包括以下特点：

  1. 使用 ReactDOM.createPortal 渲染到挂载的容器之外

  2. 提供两种使用方式，一种是传统的插槽模式：

  3. 另外一种是直接调用 Modal 的 API 进行控制：

  4. 为弹窗的显示与消失添加动画，不依赖任何库

Portals

React 中的 Portals 可以解决上面的第一个问题。官方有如下定义：

Portals provide a first-class way to render children into a DOM node that exists outside the DOM hierarchy of the parent component.

常用场景为：

A typical use case for portals is when a parent component has an overflow: hidden or z-index style, but you need the child to visually “break out” of its container. For example, dialogs, hovercards, and tooltips.

至于用法也很简单，第一个参数是一个 React.Element，第二个则是 Element 所处的容器，在该组件下放入 document.body 下。
Dialog 组件的 H5 结构分为一个 container 里面包裹着显示的内容 content ，和一个背景蒙版 mask。

display: none 之殇

然后遇到了另外一个问题，内容部分 dialog-content 动画能正常运行，但是背景蒙版却无法显示动画。
这是因为让 css 中 transition 失效的原因有：

  1. 元素在 display: none 与 display: block 中切换
  2. 元素在 height: 0 与 height: auto 中切换

解决的办法有几种：

  1. 把 display 改为 visibility 和 opacity ，然后通过 绝对定位 解决元素占用空间问题
  2. 把最后的 height 高度硬编码，如 height: 500px
  3. 把前面所说的两个动画阶段拆分成三个，如显示的过程： 容器 -> 背景 -> 内容

但是会发现在 visible = true 的时候背景蒙版是有动画效果的，但是在 visible = false 的时候却失效了。

这是因为 dialog-mask 的动画时间需要 0.3s ，可是当 visible 变量转为 false 的时候，整个 Dialog 组件已经不可见了，所以自然而然地看不见动画效果。

解决的办法就是在 visible = false 的情况下为 dialog-container 添加一个 disappear 的 class ，这个 class 会把动画延迟 0.3s，让背景蒙版动画完成后再让 Dialog 不可见。

Modal 静态方法

前面提到该组件还需要实现另外一种操作方法，即使用 Api 调用。这里直接把 show 与 hidden 函数挂载到 Modal 类上。

Modal.show

React.render

调用 Modal.show 的时候会直接根据传入 config 生成 Modal 组件。所以该方法与编写组件最大的不同就是不会返回任何 JSX 以供外面调用，需要在函数内主动地调用 ReactDom.render。

有了以上思路，先在函数中创建一个新的 <div> 节点，以供后面的 Modal 组件在上面挂载。为什么要创建一个新的节点？直接使用 React.DOM.render 在 document.body 里面不行吗？心中产生这样的疑问，于是又去翻了翻 React 的官网，上面写到：

If the React element was previously rendered into container, this will perform an update on it and only mutate the DOM as necessary to reflect the latest React element.
ReactDOM.render() controls the contents of the container node you pass in. Any existing DOM elements inside are replaced when first called. Later calls use React’s DOM diffing algorithm for efficient updates.

如果 container 里面原本已经有 React element 了，那么 React 就会利用 diff 算法将 container 内的元素更新。所以为了不影响程序内其他 React element 的呈现，最好还是把 Modal 单独放在一个 container 内。

但是此时只是将 Modal 组件隐藏了，之前创建的 container 还没有完全被销毁。
这个时候 manager.destroy 方法就起作用了，前面的代码把该方法注入到了 closeCb 中，他会在 Modal 组件隐藏后执行，这个时候会执行 ReactDom.unmountComponentAtNode 和 document.removeChild， 将 container 卸载，最后再把 modalElement 置空。

总结

通过上述 Modal 弹窗组件的实现，主要有以下收获：

  1. 理解了 React 中 Portals 和 render() 的概念及作用。
  2. 掌握了解决 css 动画失效的几种方法。
  3. 在实现组件的过程中，学会把职责拆分为 内容展示 与 显示控制。
  4. 实践了提供另外一种 API 调用方式，挂载静态函数在组件上。
  5. 在多个函数中建立联系，可以创建一个中间变量来处理
