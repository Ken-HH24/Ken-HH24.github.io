实现一个mini-react-redux

Posted on 2021-12-04 Edited on 2021-12-05

preview

上一篇完成了阅读 redux 源码并动手实现了一个 mini-redux 。redux 是一个面对 JS 的状态管理库，当投入到 React 里使用中时，如果仅从 subscribe 函数上进行展开使用，会变得十分麻烦。

react-redux 的出现使得在 React 能够更好地使用 redux ，通过 Provider 组件将整个 store 注入，然后当子组件需要使用到 store 里的数据时，通过 connect 函数，编写 mapStateToProps 和 mapDispatchToProps ，就可以将目标 store 数据注入到目标组件的 props 中。

Question

在网上经常可以看到，React 中的 useContext 和 useReducer 已经可以基本满足状态管理的需求了，那么为什么还需要 react-redux ，直接使用 useContext 并注入 redux 中的 store 不就好了吗？

首先使用 useContext 就表明不可直接将数据注入 props 中，当然使用 hooks 也有他的好处。但是由于 redux 的状态 state 是通过闭包保存在 store 里面的，所以当调用 dispatch 改变 state 时，React 的 Context API 是检测不到 store 的变化的，也无法进行更新，因此必须实现一套新的 API 供 React 使用 redux 。

Provider

首先来实现 Provider 组件，他的核心与 React 中的 Context 类似，就是注入目标数据 store ，供子组件使用。

翻开源码会发现，Provider 组件依然使用了 React Context ，主要做了以下内容：

  1. 全局通过 createContext 创建一个 ReactReduxContext 对象
  2. 在 Provider 函数组件里创建一个 subscription 对象，方便后续状态更新的派发
  3. 组件最后以 Context.Provider 渲染，注入 store 和 subscription
